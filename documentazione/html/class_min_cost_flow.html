Detailed Description

Struttura dati e metodi per un'istanza di min-cost flow con barriera.

Mantiene il grafo come lista di archi + lista di adiacenza, i vettori di costo e capacità come Eigen::VectorXd, la matrice di incidenza B (m×n) e la mappa bidirezionale undirected_edge_to_indices per lookup rapido degli archi tra due nodi.

I metodi phi, calc_gradients e calc_lengths implementano la barriera logaritmica usata dal loop MCR di AlmostLinearTime.

Note
I campi m, n, edges, adj e le matrici Eigen sono pubblici per consentire accesso diretto da Howard e calc_feasible_flow. Modificarli direttamente è sconsigliato: usare add_edge/add_vertex.

Costruttore principale: inizializza l'istanza da archi, costi e capacità.

Inizializza l'istanza calcolando tutti i campi derivati dagli archi.

Calcola automaticamente n come max(indici di nodo) + 1, costruisce la matrice di incidenza B, la mappa undirected_edge_to_indices, la lista di adiacenza adj, e i parametri U e alpha della barriera.

Parameters
edgesLista degli archi orientati (u, v).
cVettore dei costi (dimensione |edges|).
u_lowerCapacità inferiori (dimensione |edges|).
u_upperCapacità superiori (dimensione |edges|).
optimal_costCosto ottimale noto. Per istanze di max-flow è -optimal_flow.
Precondition
c.size() == u_lower.size() == u_upper.size() == edges.size()
u_lower[e] <= u_upper[e] per ogni arco e.

Operazioni eseguite

  1. Calcola n come max(tutti gli indici di nodo) + 1.
  2. Copia edges, c, u_lower, u_upper, optimal_cost.
  3. Calcola U = max(|u_upper|, |u_lower|) e alpha = 1/log₂(1000·m·U).
  4. Costruisce la matrice di incidenza B (m×n).
  5. Popola undirected_edge_to_indices con entrambe le direzioni.
  6. Costruisce la lista di adiacenza adj.
Note
c_org viene inizializzata uguale a c — copia dei costi originali prima di eventuali modifiche successive.

Eigen::VectorXd MinCostFlow::calc_lengths ( const Eigen::VectorXd &  f) const

Calcola le lunghezze sugli archi usate dall'algoritmo di Howard.

Calcola le lunghezze sugli archi per l'algoritmo di Howard.

Le lunghezze sono definite come la somma delle derivate seconde parziali della barriera rispetto a ciascun arco:

lengths_e = (u_e - f_e)^(-1-α) + (f_e - l_e)^(-1-α)

Sono sempre positive nel dominio ammissibile, garantendo che il ratio di Howard sia ben definito.

Parameters
fVettore di flusso corrente (dimensione m).
Returns
Vettore delle lunghezze di dimensione m (tutti positivi).

Le lunghezze sono proporzionali alla curvatura della barriera in f:

lengths_e = (u_e - f_e)^(-1-α) + (f_e - l_e)^(-1-α)

Sono sempre strettamente positive nel dominio ammissibile, garantendo che il ratio Σ gradients / Σ lengths sia ben definito in Howard.

Parameters

Returns

Factory method: costruisce un'istanza di min-cost flow a partire da un problema di max-flow con valore target.

Riduce un problema di max-flow a un'istanza di min-cost flow.

Aggiunge un arco virtuale (t → s) con costo -1 e capacità [0, Σ capacities], impostando optimal_cost = -optimal_flow. Gli archi originali hanno costo 0 e capacità [lower, upper].

Questa riduzione garantisce che minimizzare il costo equivalga a massimizzare il flusso sull'arco virtuale.

Parameters
edgesArchi del grafo originale.
sNodo sorgente.
tNodo pozzo.
optimal_flowValore target del flusso (stima della ricerca binaria).
capacitiesCapacità superiori degli archi originali.
lower_capacitiesCapacità inferiori (nullptr → tutti zero).
Returns
Istanza di min-cost flow pronta per il loop interior-point.

Costruzione

  • Aggiunge un arco virtuale (t → s) con costo -1 come ultimo arco.
  • Costi degli archi originali: tutti 0.
  • Capacità superiore dell'arco virtuale: Σ capacities (bound triviale).
  • optimal_cost = -optimal_flow: minimizzare il costo equivale a massimizzare il flusso sull'arco (t → s).
Note
cap_sum è calcolato come long long per evitare overflow su istanze con molti archi ad alta capacità (es. frazionari scalati).

◆ phi()

double MinCostFlow::phi ( const Eigen::VectorXd &  f) const

Valuta il potenziale Φ(f) nel punto f.

Valuta il potenziale Φ(f) = termine obiettivo + barriere di capacità.

Φ(f) = 20·m·log₂(c·f - opt) + Σ_e [ (u_e - f_e)^(-α) + (f_e - l_e)^(-α) ]
Parameters
fVettore di flusso corrente (dimensione m), strettamente interno al dominio ammissibile u_lower < f < u_upper.
Returns
Valore scalare di Φ(f). Può essere +∞ se f è sul bordo del dominio.
Precondition
(u_lower.array() < f.array()).all() && (f.array() < u_upper.array()).all()
c.dot(f) > optimal_cost

Il termine obiettivo usa log₂ (implementato come log / _LOG2) per coerenza con il parametro alpha che è definito rispetto a log₂. Le barriere sono somme di potenze negative: divergono a +∞ quando f si avvicina ai bordi u_lower o u_upper.

Parameters
fFlusso corrente, strettamente ammissibile: u_lower < f < u_upper.
Returns
Valore scalare Φ(f).
