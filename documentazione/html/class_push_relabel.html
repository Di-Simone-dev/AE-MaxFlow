Detailed Description

Implementazione dell'algoritmo Push-Relabel (Goldberg & Tarjan, 1988).

Calcola il flusso massimo su un grafo orientato con capacità degli archi di tipo long long (modalità intera) oppure double (modalità virgola mobile). Il tipo si sceglie tramite il costruttore appropriato.

Uso tipico (modalità intera)

PushRelabel::IntGraph g;
g[{0, 1}] = 10;
g[{0, 2}] = 5;
g[{1, 3}] = 8;
g[{2, 3}] = 7;
PushRelabel pr(g);
auto result = pr.max_flow(0, 3);
long long flow = std::get<long long>(result); // 13

PushRelabel::PushRelabel ( const IntGraph &  graph)
explicit

Costruisce un oggetto PushRelabel in modalità intera.

Costruisce il grafo residuo interno a partire da graph. Per ogni arco (u, v, cap) vengono aggiunti:

  • un arco diretto con capacità cap;
  • un arco inverso con capacità 0 (per il flusso di ritorno).

Tutti i nodi presenti come estremi degli archi vengono registrati automaticamente; non è necessario dichiararli in anticipo.

Parameters
graphMappa degli archi con le rispettive capacità intere.
Postcondition
max_flow() restituirà un std::variant con long long.

Il costruttore esegue tre passi:

  1. Raccolta dei nodi: scorre tutti gli archi di graph e registra ogni estremo in _index (se non già presente), assegnandogli un indice interno progressivo 0-based. L'ordine di visita dipende dall'iterazione sulla unordered_map, quindi non è deterministico, ma questo non influisce sulla correttezza dell'algoritmo.
  2. Allocazione della lista di adiacenza: ridimensiona _adj_int a _n liste vuote.
  3. Inserimento degli archi residui: per ogni arco (u → v, cap):
    • aggiunge a _adj_int[u] l'arco diretto con capacità cap;
    • aggiunge a _adj_int[v] l'arco inverso con capacità 0. I campi rev dei due archi si puntano a vicenda, consentendo aggiornamenti O(1) durante le operazioni di push.

Costruisce un oggetto PushRelabel in modalità double.

Identico al costruttore intero, ma le capacità sono di tipo double. I confronti con zero vengono effettuati con la soglia PushRelabel::EPS per evitare push infiniti dovuti a errori di arrotondamento.

Parameters
graphMappa degli archi con le rispettive capacità floating-point.
Postcondition
max_flow() restituirà un std::variant con double.

Identico al costruttore per IntGraph, ma opera su _adj_dbl e imposta la capacità degli archi inversi a 0.0 (double) anziché 0LL. Il flag _is_double viene impostato a true per indirizzare le chiamate successive a _max_flow_dbl tramite il dispatcher max_flow().

std::variant< long long, double > PushRelabel::max_flow ( int  source,
int  sink 
)

Calcola il flusso massimo da source a sink.

Esegue l'algoritmo Push-Relabel con FIFO, gap heuristic e current-arc sul grafo residuo costruito dal costruttore. Il grafo residuo viene modificato in-place: le capacità riflettono il flusso inviato dopo il ritorno di questa funzione.

Il tipo del valore restituito dipende dal costruttore usato:

Costruttore Tipo nel variant Come accedere
IntGraph long long std::get<long long>(res)
DblGraph double std::get<double>(res)
Parameters
sourceIdentificatore del nodo sorgente (deve essere nel grafo).
sinkIdentificatore del nodo pozzo (deve essere nel grafo).
Returns
Valore del flusso massimo come std::variant<long long, double>.
Exceptions
std::out_of_rangeSe source o sink non sono presenti nel grafo.
Note
Se source == sink, il flusso restituito è 0 (senza eseguire l'algoritmo).

Il metodo:

  1. Verifica che source e sink esistano in _index, lanciando std::out_of_range in caso contrario.
  2. Gestisce il caso degenere source == sink restituendo 0 senza eseguire l'algoritmo.
  3. Delega a _max_flow_int() o _max_flow_dbl() in base al flag _is_double impostato dal costruttore.

Il risultato è incapsulato in un std::variant<long long, double> per unificare l'interfaccia pubblica indipendentemente dalla modalità.
