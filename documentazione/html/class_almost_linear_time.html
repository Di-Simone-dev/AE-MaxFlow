Detailed Description

Solver per il problema di massimo flusso con complessità quasi-lineare.

Implementa un algoritmo di tipo interior-point che:

  1. Esegue una ricerca binaria sul valore del flusso ottimale.
  2. Per ogni candidato, costruisce un'istanza di min-cost flow tramite MinCostFlow::from_max_flow_instance.
  3. Calcola un flusso ammissibile iniziale con almost_linear::calc_feasible_flow.
  4. Itera aggiornamenti del flusso lungo il ciclo di minimum cycle ratio (trovato con l'algoritmo di Howard) fino a convergenza del potenziale Φ(f).
Note
Il grafo è rappresentato come mappa di archi orientati con capacità intere a 64 bit per evitare overflow su istanze con capacità elevate (es. capacità frazionarie scalate prima della chiamata).
Warning
L'algoritmo assume che il grafo sia connesso e che source ≠ sink. Grafi con source isolato producono flusso 0 senza eccezioni.

Calcola il valore del massimo flusso tra source e sink.

Calcola il valore del massimo flusso tramite ricerca binaria.

Esegue una ricerca binaria sull'intervallo [0, Σ cap(source, v)], invocando max_flow_with_guess per ogni candidato mid. La ricerca termina quando low == high, restituendo il massimo valore di flusso raggiungibile.

Complessità: O(log(U) · T_guess), dove U è la somma delle capacità uscenti da source e T_guess è il costo di una singola chiamata a max_flow_with_guess.

Parameters
sourceIndice del nodo sorgente (0-based).
sinkIndice del nodo pozzo (0-based).
Returns
Valore intero del massimo flusso da source a sink.

Il limite superiore della ricerca è la somma delle capacità degli archi uscenti da source. Ad ogni iterazione si testa il valore mid:

  • Se max_flow_with_guess restituisce un flusso < mid, il valore mid non è raggiungibile → si abbassa high.
  • Altrimenti → si alza low.

La ricerca termina quando low == high, ovvero quando l'intervallo si è ristretto a un singolo valore.

Risolve il max-flow con una stima fissata del valore ottimale.

Costruisce l'istanza di min-cost flow corrispondente al flusso target optimal_flow, calcola un flusso ammissibile iniziale e itera aggiornamenti lungo il ciclo di minimum cycle ratio fino a convergenza.

Il loop di ottimizzazione termina quando:

  • c·f - optimal_cost < threshold (convergenza), oppure
  • Φ(f) non è finito dopo un aggiornamento (bordo del dominio).

Ad ogni iterazione vengono verificate (via assert) le invarianti:

  • Conservazione del flusso: ||B^T f||_∞ < 1e-10
  • Ciclo con ratio negativo: min_ratio < 0
  • Decrescita stretta del potenziale: Φ(f_new) < Φ(f_old)

sourceIndice del nodo sorgente (0-based).

lower_capacitiesCapacità inferiori sugli archi (opzionale, default nullptr → tutti zero).

Coppia (valore del flusso sull'arco virtuale s→t, vettore dei flussi sugli archi originali arrotondati).

Dettagli implementativi

Parametri del loop:

  • threshold = 1e-5: tolleranza sulla convergenza di c·f - optimal_cost.
  • kappa = 0.9999: fattore di damping per lo step-size η.
  • upscale = 1000.0: fattore di scala applicato all'aggiornamento del flusso per mantenere il flusso lontano dai bordi del dominio ammissibile di Φ.

Step-size: η = -(κ²) / (50 · ∇Φ · d), dove d è il vettore del ciclo critico. Moltiplicato per upscale garantisce passi di dimensione adeguata.

Invarianti verificate ad ogni iterazione (via assert):

  • Conservazione del flusso: ||B^T f||_∞ < 1e-10
  • Ciclo critico con ratio negativo: min_ratio < 0
  • Decrescita stretta del potenziale: Φ(f_new) < Φ(f_old)

Coppia (flow_value, edge_flows):
  • flow_value: valore intero del flusso sull'arco virtuale t→s.
  • edge_flows: vettore dei flussi arrotondati sugli archi originali.

Aggiunge un arco virtuale source→sink con costo -1 e capacità optimal_flow, impostando optimal_cost = -optimal_flow come lower bound sul costo.

calc_feasible_flow aggiunge archi ausiliari da/verso un nodo fittizio v* per bilanciare gli squilibri di flusso, producendo un punto interno al dominio ammissibile di Φ(f).

< Tolleranza di convergenza su c·f - opt

< Damping factor per lo step-size η

< Scala dell'aggiornamento del flusso

Howard restituisce il ciclo con ratio minimo (negativo) e il vettore indicatore del ciclo nel grafo di policy.

η garantisce che Φ(f + η·upscale·d) < Φ(f) per la struttura self-concordante della barriera logaritmica.
